```markdown
# Along Working Thread

Along Working Thread is a continuity-aware co-creator for active Codex sessions.

It helps Codex carry project judgment across sessions, notice meaningful drift, and draft wrap-ups with your confirmation.

## What It Does

- Restores relevant Working Thread context when an Along project session resumes.
- Keeps ordinary small requests quiet and direct.
- Adds a light note for nearby future-direction exploration.
- Pauses before high-impact direction switches and asks for confirmation.
- Drafts bounded Working Thread updates after confirmed direction switches.
- Requires user confirmation before durable Working Thread write-back.

## What It Does Not Do

- It is not a background autonomous agent.
- It does not watch files or time when Codex is closed.
- It does not send notifications.
- It does not provide always-on desktop presence.
- It does not implement Along Core, MCP, Hermes, Claude Code adapters, Memory v2, relationship modes, delegation, or write delegation.
- It does not silently create or update durable Working Thread records.

## Working Threads

A Working Thread is a judgment container for an unfinished direction, doubt, or creative line that should affect future decisions.

It is not a chat transcript, todo list, issue ticket, or generic memory.

The useful distinction is:

```text
Chat = where conversation happens.
Working Thread = what important unfinished judgment the conversation carries forward.
```

## Turn-Bound Self-Initiation

This plugin provides turn-bound self-initiation. Codex can act proactively inside an active user-agent session by restoring context, noticing drift, or suggesting wrap-up.

It does not provide background self-initiation. If no active agent session exists, the plugin does not observe, wake, notify, or intervene.

## Starter Prompts

```text
Resume the current Working Thread.
Help me wrap up this phase.
Check whether this direction drifts from our thread.
```
