# Navi 0.1.0-alpha.3

Navi `0.1.0-alpha.3` is a GitHub source-alpha maintenance checkpoint for prompt-language following in Navi maps.

Use this release if you want the current source package, the `navi init` project-local setup path, and the latest language-following behavior for multilingual target projects.

Wait for a later release if you need npm distribution, public marketplace installation, global plugin installation, one-click sync, runtime UI, background watching, notifications, always-on presence, or adapters for agents outside Codex.

## Highlights

- Keeps `HezLUO/navi` as the canonical standalone Navi alpha repository while Along remains the broader long-term companion-layer vision.
- Clarifies that Navi map responses follow the user's current prompt language by default.
- Clarifies that saved Project Map or Rhythm Map records are source evidence, not the answer-language selector.
- Updates the repo skill, packaged plugin skill, project-map docs, and `navi init` generated trigger block with the language-following rule.
- Keeps the current alpha surface unchanged: skill/plugin behavior, project-local docs, and `navi init`.

## Verify From Source Package

```bash
git clone https://github.com/HezLUO/navi.git
cd navi
npm install
npm run verify:plugin-package
npm run typecheck
npm test
```

## Try Project-Local Setup

Preview the target-project setup first:

```bash
npm run navi -- init --target /path/to/target-project
```

If the preview is acceptable, apply it explicitly:

```bash
npm run navi -- init --target /path/to/target-project --write
```

`navi init` prepares target-project files such as `AGENTS.md` and `docs/along/project-maps/navi-project-map.md`. It is dry-run by default and does not install the global Codex plugin or skill. For setup details, use `docs/along/project-maps/navi-project-init.md` and `docs/along/project-maps/navi-project-trigger-template.md`.
