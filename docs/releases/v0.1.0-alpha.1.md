# Navi 0.1.0-alpha.1

Navi `0.1.0-alpha.1` clarifies the product hierarchy after the first alpha source release.

Navi is now treated as the independent open-source alpha product. Along remains the broader long-term companion-layer vision.

## Who Should Use This Release

Use this release if you want to inspect Navi's current source package, test Navi supervision behavior in active Codex sessions, or give feedback on whether Progress/Rhythm Maps and Challenge Layer behavior help non-expert users steer expert-agent work.

Wait for a later release if you need global plugin installation, one-click sync, npm distribution, public marketplace installation, runtime UI, background watching, notifications, or adapters for agents outside Codex.

## Highlights

- Clarifies that Navi is the current independent V1 product surface from the Along vision, not the full Along roadmap.
- Keeps the V1 alpha behavior centered on Progress Maps, Rhythm Maps for flowing projects, and Challenge Layer behavior.
- Preserves the `along-working-thread` package and skill ids as alpha compatibility names.
- Keeps npm publication, public marketplace publication, global plugin installation, one-click sync, runtime UI, background watcher behavior, and cross-agent adapters outside the current alpha scope.

## Verify From Source Package

```bash
git clone https://github.com/HezLUO/navi.git
cd navi
npm install
npm run verify:plugin-package
npm run typecheck
npm test
```

After source verification, preview the project-local setup in a real target project:

```bash
npm run navi -- init --target /path/to/target-project
```

If the preview is acceptable, apply it explicitly:

```bash
npm run navi -- init --target /path/to/target-project --write
```

`navi init` prepares target-project files such as `AGENTS.md` and `docs/along/project-maps/navi-project-map.md`. It is dry-run by default and does not install the global Codex plugin or skill. For setup details, use `docs/along/project-maps/navi-project-init.md` and `docs/along/project-maps/navi-project-trigger-template.md`.

## Feedback We Want

- Do ordinary progress, next-step, confusion, continue, and plan-reliability questions trigger a useful Navi map?
- Do flowing projects use Rhythm Maps instead of misleading completion bars?
- Does Challenge Layer behavior catch weak assumptions or self-certifying momentum without becoming constant critique?
- Do narrow factual checks and clear execution requests stay quiet?
- Does the `along-working-thread` compatibility name confuse installation or source review?

This is a GitHub source pre-release for developers and early testers.
